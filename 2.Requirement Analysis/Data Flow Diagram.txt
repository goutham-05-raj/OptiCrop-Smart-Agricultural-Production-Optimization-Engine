1. Data Flow DFD Level 0 (Context Diagram):
   [User (Farmer/Researcher)] --(Soil & Weather Inputs)--> [Opticrop Engine] --(Crop Recommendations & Telemetry Charts)--> [User]

2. Data Flow DFD Level 1 (Process Diagram):
   - Process 1.0 (Input Validation): Receives form data (N, P, K, pH, temp, humidity, rainfall). Validates bounds. Passes sanitized values to Process 2.0.
   - Process 2.0 (Model Predict / Inference):
     * Invokes StandardScaler (scaler.pkl) to scale continuous inputs.
     * Passes scaled metrics to Random Forest Classifier (best_model.pkl) to recommend crop.
     * Passes scaled metrics to K-Means Clustering (kmeans_model.pkl) to identify soil category.
     * Encodes categorical fields (location, season) and passes features to Random Forest Regressor (yield_model.pkl) to predict yield.
   - Process 3.0 (Nutrient Action Engine): Compares predicted crop NPK baseline to input soil values, calculating deficiencies/excesses.
   - Process 4.0 (Logging / Storage): Appends inputs, predictions, yield, and soil cluster to 'datasets/user_predictions.csv'.
   - Process 5.0 (Rendering): Collects predictions, NPK corrections, and metrics graphs to display on 'result.html' and 'dashboard.html'.
