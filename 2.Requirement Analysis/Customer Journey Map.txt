This map outlines the path Rajesh takes when interacting with the Opticrop system, from soil sampling to harvesting:

1. Sampling Phase:
 - Actions: Collects soil, obtains NPK and pH levels from local testing lab.
 - Touchpoints: Local agricultural testing lab.
 - Pain Points: Slow lab reports, manual record-keeping.
 - Opportunities: Provide advice on how to sample soil correctly in the application UI.

2. Accessing Phase:
 - Actions: Navigates to Opticrop portal using a web browser.
 - Touchpoints: Web Portal Login.
 - Pain Points: Unstable mobile network, authentication hurdles.
 - Opportunities: Create lightweight glassmorphic responsive UI pages optimized for slow networks.

3. Input Phase:
 - Actions: Inputs N, P, K, pH, temperature, rainfall, and season into the form.
 - Touchpoints: Fields Configurator Form.
 - Pain Points: Form confusion, units mapping errors.
 - Opportunities: Add input validation, tooltips, and default placeholder bounds.

4. Predict Phase:
 - Actions: Submits inputs, gets crop recommendation, expected yield, and soil cluster.
 - Touchpoints: Results Presentation Panel.
 - Pain Points: Slow calculations, complex statistics.
 - Opportunities: Return instant visual indicators and recommendation confidence metrics.

5. Action Phase:
 - Actions: Applies prescribed fertilizers, plants crop.
 - Touchpoints: Field application.
 - Pain Points: Lack of step-by-step guidance.
 - Opportunities: Generate clear NPK corrective instructions in a simple checklist.

6. Harvest Phase:
 - Actions: Grows crop, records final harvest yield.
 - Touchpoints: Yield Log Form.
 - Pain Points: Manual data entry, lack of incentive.
 - Opportunities: Incorporate harvest results to update dataset models for future cycles.
